# Security Policy

VSCode Stimulator can execute Python code and shell commands through its backend. Treat it like a real development environment.

## Recommended Safe Usage

1. Run private or sensitive work locally or on a protected server.
2. Do not expose the backend publicly without authentication.
3. Do not run code from people you do not trust.
4. Do not paste GitHub tokens into screenshots, public videos or demo sessions.
5. Keep `.env` out of GitHub.
6. Rotate any token that may have been exposed.
7. Use GitHub Pages or Cloudflare Pages only for static UI demos.
8. Use Render or Termux for backend execution.

## No Paid AI API Runtime

The application does not require OpenAI, Gemini, Claude or any paid AI API key to run. This reduces cost and reduces the risk of leaking paid API credentials.
